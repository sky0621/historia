# historia MVPタスク分解

## 使い方
- このファイルは MVP 実装用の実行タスク一覧。
- 上から順に着手する前提で並べている。
- 各タスクは「目的」「作業」「完了条件」「依存」を持つ。
- 実装中は必要に応じて `Status` を `todo` / `doing` / `done` に更新して運用する。

## T001 プロジェクト初期化
- Status: `todo`
- 目的: 実装の土台を作る。
- 作業:
  - Next.js App Router のプロジェクト構成を作成する。
  - TypeScript、Lint、基本ディレクトリ構成を整える。

  - 開発起動手順を README に反映する。
- 完了条件:

  - DB 接続設定が読み込まれる。
  - 初回マイグレーション実行手順が確立している。
- 依存: なし

## 実装上の注意
- `TimeExpression` は途中で仕様変更すると全画面に波及するので、早い段階で固定する。
- 多対多リンクが多いため、保存 API はモデルごとにバラバラにせず、更新戦略を揃える。
- MVP では全体グラフビューを作らない。詳細画面上の関連リンク整備を優先する。
- 戦争・乱の結果モデルは簡易版に留め、UI も過剰に複雑化しない。
