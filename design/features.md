# historia 実装済み機能一覧

最終更新時点で、`Sprint 1` から `Sprint 7` までの初版実装が入っている。

## 1. 基盤
- `Next.js 15 + TypeScript + Tailwind CSS 4 + Drizzle + SQLite` の構成で動作する。
- ローカル実行前提で、認証なしの単一ユーザー利用を想定している。
- `TimeExpression` を共通時間表現として持ち、`BCE/CE`、推定年、範囲年、年未詳を扱える。
- `seed`、`migration`、`lint`、`typecheck`、`test`、`build` が通る構成になっている。

## 2. 主要ドメイン CRUD

### 2.1 イベント
- イベントの一覧、詳細、新規作成、編集、削除ができる。
- `general / war / rebellion / civil_war` の種別を持てる。
- 人物、国家、王朝、時代区分、宗教、宗派、地域、タグを関連付けできる。
- `before / after / cause / related` のイベント間関係を登録できる。

### 2.2 人物
- 人物の一覧、詳細、新規作成、編集、削除ができる。
- 生没年、別名、地域、宗教、宗派、時代区分を管理できる。
- 役職履歴を複数件持てる。
- 役職は国家または王朝に紐づけられる。

### 2.3 国家・王朝
- 国家の一覧、詳細、新規作成、編集、削除ができる。
- 王朝の一覧、詳細、新規作成、編集、削除ができる。
- 国家と王朝は開始・終了年、別名、メモ、地域を持てる。
- 王朝は国家に必須で所属する。

### 2.4 時代区分・カテゴリ
- 時代区分カテゴリの一覧、詳細、新規作成、編集、削除ができる。
- 時代区分の一覧、詳細、新規作成、編集、削除ができる。
- 時代区分はカテゴリ、対象国家、対象地域名、関連地域を持てる。

### 2.5 宗教・宗派
- 宗教の一覧、詳細、新規作成、編集、削除ができる。
- 宗派の一覧、詳細、新規作成、編集、削除ができる。
- 宗教・宗派は開始・終了年、別名、説明、メモ、地域、開祖を持てる。
- 宗派は宗教に必須で所属する。

### 2.6 地域・タグ・出典
- 地域の一覧、詳細、新規作成、編集、削除ができる。
- 地域は親子階層を持てる。
- タグの一覧、詳細、新規作成、編集、削除ができる。
- 出典の一覧、詳細、新規作成、編集、削除ができる。
- 出典に対して引用を追加・編集できる。

## 3. 戦争・乱
- 戦争や乱を期間付きイベントとして管理できる。
- 参加勢力を `人物 / 国家 / 宗教 / 宗派` から複数登録できる。
- 参加勢力には `attacker / defender / leader / ally / other` を持てる。
- 結果として勝者側、敗者側、結果要約、補足メモを登録できる。
- 勝者側・敗者側には参加勢力そのものを紐づけられる。

## 4. 一覧検索・探索

### 4.1 一覧検索
- イベント一覧は `キーワード / 種別 / 関係種別 / 人物 / 国家 / 王朝 / 宗教 / 宗派 / 地域 / 時代区分 / タグ / 年代範囲 / 並び順` で絞り込める。
- 人物一覧は `地域 / 宗教 / 宗派 / 時代区分 / 国家 / 王朝 / 役職あり` で絞り込める。
- 国家、王朝、宗教、宗派、時代区分、時代区分カテゴリ、地域、タグもそれぞれ実フィルタを持つ。

### 4.2 相互リンク
- 主要詳細画面は関連主体や関連イベントへのリンクを持つ。
- ヘッダ件数や一覧件数から、絞り込み済み一覧へ直接遷移できる。
- 関連イベントカードには時間・種別・主要関連主体の要約が表示される。

## 5. 可視化
- `/graph/events` でイベント関係グラフを表示できる。
- イベント関係に加え、人物・国家・宗教・地域を補助ノードとして表示できる。
- `/timeline` でイベント、時代区分、国家、王朝、宗教、宗派を同一タイムライン上に表示できる。

## 6. データ運用
- `/manage/data` で運用機能にアクセスできる。
- JSON export ができる。
- JSON import ができる。
- import は preview 付きで、重複候補を表示する。
- `Event` と `Person` の CSV export ができる。
- 主要詳細画面では出典と変更履歴を確認できる。

## 7. 出典・履歴
- `Source` を書誌情報または URL として管理できる。
- `Citation` を `Event / Person / Polity / HistoricalPeriod / Religion` に紐づけられる。
- `Event / Person / Polity / HistoricalPeriod` は変更履歴を保存する。
- 詳細画面で直近の変更履歴を確認できる。

## 8. 高度モデル化

### 8.1 国家変遷
- `PolityTransition` を登録できる。
- `split / merge / rename / succession / other` を扱える。
- 国家詳細で変遷を表示できる。

### 8.2 王朝継承
- `DynastySuccession` を登録できる。
- 国家詳細と王朝詳細で継承関係を表示できる。

### 8.3 宗派分派
- `Sect.parentSectId` を持ち、親宗派と子宗派を表現できる。
- 宗派詳細で親宗派と子宗派を表示できる。

### 8.4 地域関係
- `RegionRelation` を登録できる。
- `contains / adjacent / cultural_sphere` を扱える。
- 地域詳細で関係を表示できる。

### 8.5 時代区分関係
- `HistoricalPeriodRelation` を登録できる。
- `contains / overlaps / succeeds` を扱える。
- 時代区分詳細で関係を表示できる。
